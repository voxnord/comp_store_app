# Система учета и продаж компьютерной техники

Веб-приложение для автоматизации бизнес-процессов компьютерного магазина. Проект включает в себя полноценную систему управления складом, сотрудниками и продажами (Admin/ERP панель), а также публичную интернет-витрину для покупателей.

## Описание функционала

Проект разделен на две основные части: Панели сотрудников (Employee Panel) с доступом на основе ролевой модели и витрина товаров (Shop) для клиентов.

### Панели сотрудников:
- Администратор: Полный контроль над системой. Управление сотрудниками и клиентской базой, создание/редактирование ролей, управление иерархией категорий товаров, просмотр расширенной статистики и истории всех продаж с возможностью печати чеков;
- Менеджер: Управление товарной номенклатурой (CRUD операций, установка минимальных остатков, цен). Доступ к POS-терминалу для оформления офлайн-продаж, ведение базы клиентов.

### Витрина товаров:
- Каталог и поиск: Просмотр товаров с проверкой наличия в реальном времени. Интеллектуальный поиск по названию или артикулу;
- Корзина: Добавление товаров в корзину с валидацией доступных остатков на складе;
- Личный кабинет: Регистрация, авторизация клиентов и оформление онлайн-заказов (заказы автоматически попадают в общую базу со статусом "новый").

## Стек технологий

- Backend: Python 3.8+, Flask 2.3+ (паттерн Application Factory, Blueprints).
- База данных: PostgreSQL 12+ (взаимодействие через psycopg2).
- Frontend: HTML5, CSS3, JavaScript (включая скомпилированный TypeScript для логики POS-терминала), Bootstrap 5, Jinja2.
- Безопасность: Хеширование паролей (werkzeug.security), декораторы для ограничения доступа по ролям.

## Структура базы данных

Схема БД спроектирована с учетом строгой ссылочной целостности и использует индексы для оптимизации поиска.

| Таблица | Описание | Ключевые поля и связи |
| :--- | :--- | :--- |
| **role** | Роли пользователей в системе | Название, права доступа (JSONB) |
| **employee** | Данные сотрудников | ФИО, должность, контакты, дата найма |
| **user** | Учетные записи (Auth) | Логин, хеш пароля, FK на `role`, `employee` |
| **customer** | База покупателей (Физ. лица) | ФИО, контакты, согласие на ПДн, FK на `user` |
| **category** | Категории товаров | Название, иерархия (FK `parent_id` на `category`) |
| **product** | Товарная номенклатура | Артикул, цена, остаток, мин. уровень, FK на `category` |
| **sale** | Заказы и офлайн-чеки | Номер, дата, сумма, статус, метод оплаты |
| **sale_item** | Позиции внутри чека/заказа | Цена на момент продажи, кол-во, FK на `sale`, `product` |

## Ключевые API и маршруты

Приложение разделено на независимые Blueprints: `main`, `admin`, `manager`, `shop`.

| Маршрут | Метод | Blueprint | Описание |
| :--- | :---: | :---: | :--- |
| `/shop/` | `GET` | **shop** | Главная страница витрины, каталог товаров |
| `/shop/cart` | `GET`, `POST` | **shop** | Просмотр корзины и оформление онлайн-заказа |
| `/manager/pos` | `GET` | **manager** | Интерфейс POS-терминала для сотрудников |
| `/manager/api/create-order` | `POST` | **manager** | API (JSON) для оформления заказа из POS-терминала |
| `/manager/products` | `GET`, `POST` | **manager** | Управление товарами (CRUD) |
| `/admin/users` | `GET`, `POST` | **admin** | Управление учетными записями, блокировка |
| `/admin/api/sale-details/<id>` | `GET` | **admin** | API (JSON) для получения расшифровки чека |
| `/api/update-sale-status/<id>` | `POST` | **admin** | API обновления статуса заказа (AJAX) |

## Инструкция по установке и запуску

### 1. Подготовка базы данных

Убедитесь, что у вас установлен PostgreSQL. Создайте базу данных и инициализируйте структуру:

```
createdb comp_store_ais
psql -h localhost -U postgres -d comp_store_ais -f create_database.sql
```

Скрипт автоматически создаст тестового пользователя admin с паролем admin123.

### 2. Клонирование и настройка окружения

```
# Клонирование репозитория
git clone <https://github.com/voxnord/comp_store_app>
cd <comp_store_app>

# Создание и активация виртуального окружения
python -m venv venv
# На Windows: venv\Scripts\activate
# На Linux/Mac: source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 3. Конфигурация

В файле app/config.py задайте параметры подключения к БД (или используйте переменные окружения):

```python
DB_HOST = 'localhost'
DB_PORT = 5432
DB_NAME = 'имя_базы_данных'
DB_USER = 'имя_пользователя'
DB_PASSWORD = 'ваш_пароль'
```
### 4. Запуск сервера

```
python run.py
```

Приложение будет доступно по адресам:
- Витрина магазина: http://localhost:5000/shop/
- Вход для сотрудников: http://localhost:5000/login

## Пример использования

1. Регистрация товара: Администратор или менеджер заходит в панель управления (/manager/products), нажимает "➕ Регистрация нового товара", вводит артикул (например, INT-13900), название, цену и текущий остаток на складе.
2. Продажа через POS (Офлайн): Менеджер переходит в раздел "Терминал продаж" (/manager/pos). Кликает "В корзину" на нужных товарах. Выбирает клиента из базы или оставляет "Розничный покупатель", выбирает тип оплаты и нажимает "ОФОРМИТЬ ПРОДАЖУ".
3. Обработка заказа: Система (Sale.create_transaction) открывает транзакцию в БД, записывает чек, автоматически списывает количество товара со склада и перенаправляет на страницу генерации чека.
4. Онлайн-покупка: Покупатель заходит на витрину (/shop/), кладет товар в корзину, авторизуется и оформляет заказ. Менеджер в панели управления видит новый заказ со статусом "Новый" в истории продаж.
